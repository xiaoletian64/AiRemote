# HID 噪声误输入修复设计

## 目标

遥控器的圆盘与圆环数据报告不得再被解释为方向、确认或其他已映射按键；正常按键报告、圆盘滚动和语音流程保持不变。

## 根因

`Engine.handleHIDReport(_:)` 在区分报告格式前调用 `XiaomiRemoteHIDParser.usage(in:known:)`。该解析器会扫描任意原始字节对和单字节兜底匹配。圆盘 `0x03` 报告及 10 字节圆环报告的位移/触摸数据一旦包含已知 usage，例如 `0x28`，就会被提前派发为真实按键。

## 方案

在 `handleHIDReport(_:)` 开头按报告类型建立安全边界：

1. `0x03`、7 字节圆盘报告只进入既有 `RingScrollDecoder`。圆盘功能关闭时静默忽略；开启时仅在连续稳定帧达到阈值后发送滚动。
2. `0x01`、10 字节报告暂不按长度一概丢弃：它可能与真实键盘报告共用格式，继续保持既有路径，等待现场诊断样本后再精确收紧。
3. 其余报告维持原有“已知 usage → release → 未识别诊断”的顺序，不改变合法按键的映射语义。

## 验收与测试

- 含 `0x28` 等已知 usage 值的 `0x03` 圆盘报告不会返回按键 usage，也不会触发按键处理。
- 非 `0x03` 的普通按键报告仍保留按键解析资格。
- `RingScrollDecoder` 的稳定三帧滚动行为和现有遥控器输入安全测试仍通过。
