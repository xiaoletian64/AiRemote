# 遥控器输入安全与圆盘滚动设计

## 目标

在小米蓝牙遥控器 2 Pro 静置于桌面时，不让偶发 HID 报告影响 Mac 键盘；同时把其圆盘的稳定转动默认映射为快速页面滚动。

## 已验证事实

- 最新应用日志记录过 `150ms` 内三种按键切换的抖动，且出现 Back、OK、Home 与 Power 报告。
- Mac HID 设备枚举误纳入了 `Apple Internal Keyboard / Trackpad`，不应将其注册为遥控器。
- 2 Pro 的圆盘产生 `Report ID 0x03`、长度为 7 字节的报告；现有实现明确将其标注为鼠标位移与滚轮报告，但为避免误滚动而丢弃。

## 方案比较

1. 直接按猜测的字节位解析圆盘并启用滚动：最快，但方向或步长错误会导致放桌面时狂滚。
2. 仅提供人工学习模式：可靠，但用户每次换设备都需要额外操作。
3. 采样校准后默认滚动（采用）：内置短时采样和稳定性门槛，首次实际圆盘报告足以确认方向；通过校准后自动启用快速滚动，未知或不稳定报告保持静默。

## 设计

### Mac 应用

- HID 枚举仅接受 Xiaomi `0x2717` 和 Lenovo `0x17EF` 的已知遥控器；产品名匹配必须同时满足“非 Apple + 遥控器关键词”，不能以零值 VID 放行内建键盘。
- 将 `0x03` 七字节圆盘报告送入独立解析器。候选滚轮变化必须在相邻帧间隔不超过 `80ms`、连续至少 `3` 帧、同一字节位保持同一非零变化方向；否则视为静置噪声并忽略。每次稳定序列最多发 `6` 个 line，连续滚动最小间隔 `15ms`，避免页面失控。
- 第一次接收到 `0x03` 报告时，自动开启 `2s` 校准窗口并写原始字节。只有存在唯一符合上述稳定序列的字节位时才确定滚动轴和符号；没有唯一候选则保持静默并记录“校准不足”。校准仅保存在内存：应用重启、BLE 断连、超过 `5s` 没有圆盘帧、或者已校准字节位连续 `3` 帧违反方向一致性时，均重置并重新校准。通过校准后以有界 `CGEvent` scroll line 默认用于页面快速滚动。
- 持久日志逐行写入 ISO 时间戳、原始圆盘字节、已判定方向、过滤原因和修饰键状态；方便验证左 Command 是否有 down 无 up。

### 验收标准
1. 内建键盘不再出现在“遥控设备”日志中。
2. 模拟 BLE 断开后，USB 键盘状态报告为全零，左/右 Command 均未保持。
3. 同一按键少于 `35ms` 的完整 down/up 孤立脉冲不会触发映射；正常长按和间隔至少 `45ms` 的连续方向键仍可用。
4. 2 Pro 圆盘连续至少 3 帧、每帧间隔不超过 80ms 的稳定转动可快速滚动页面，反向稳定序列反向滚动；静置 30 秒无滚动事件。
5. 日志能够关联每个 `0x03` 原始报告与滚动/忽略判定。

## 验证方式

- 用 Swift 纯逻辑测试覆盖设备过滤、按键短脉冲、圆盘帧分类、稳定门槛与方向。
- 只构建 Mac 应用。
- 连接真实 2 Pro：分别静置、按方向键、转圆盘顺逆各三圈、按住/松开左右 Command 组合，并检查持久日志和页面行为。
